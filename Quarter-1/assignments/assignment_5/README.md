Turn what you already do into 3-5 focused, reusable capabilities that replace parts of your own job and could help others with similar work.

This assignment helps you identify and codify your daily work into Agent Skills that can automate repetitive cognitive tasks, save time, and reduce mental load.

## üìã ACTIONS

### 01. Describe Your Daily/Weekly Work to Claude Code

**In your project folder with Claude Code running:**

- Use Claude in your terminal (within the `assignment-5-skills` project folder)
- Describe your typical daily or weekly work routine to Claude
- Include all tasks, decisions, analyses, and coordination activities
- Be specific about the types of work you do regularly
- Ask Claude to help you document this in a file (e.g., `my-work-description.md`)

### 02. Identify Repeatable Cognitive Tasks

From your work description, identify tasks that involve:
- **Decisions:** Making choices based on criteria or rules
- **Writing:** Creating content, reports, emails, documentation
- **Analysis:** Reviewing data, comparing options, evaluating outcomes
- **Coordination:** Managing workflows, organizing information, scheduling

Focus on tasks that:
- Happen frequently (daily or weekly)
- Follow similar patterns each time
- Require cognitive effort but could be automated
- Have clear inputs and outputs

### 03. Implement 3-5 Narrow, Measurable Skills

**Using Claude in your project folder, create skills that:**

1. **You actually use** - Solves a real problem in your workflow
2. **Saves time or reduces mental load** - Provides measurable value
3. **Could be reused by someone like you** - Has value beyond your personal use

**For each skill:**
- Work with Claude to create the skill folder structure in your project
- Use Claude to help write the SKILL.md file and any supporting code
- Test each skill with Claude to ensure it works correctly
- Keep all skills organized in your `assignment-5-skills` project folder

Each skill should be:
- **Narrow in scope** - Focused on one specific task
- **Measurable** - You can quantify the time saved or quality improved
- **Reusable** - Others in similar roles could benefit

## ‚ö†Ô∏è CONSTRAINTS

### Skills Must Be Small
- **100-900 lines of code (LOC) is fine**
- Keep skills focused and manageable
- Avoid creating monolithic skills that do too much

### Each Skill Must Have One Clear Outcome
- One skill = one specific result
- Clear success criteria
- Predictable output format

### Each Skill Must Be Measurable
- Define metrics for success
- Track time saved, errors reduced, or quality improved
- Be able to demonstrate the value

## üí° Tips for Success

1. **Start Small:** Begin with one skill that solves a clear, frequent problem
2. **Be Specific:** The more specific your skill, the more valuable it will be
3. **Measure Everything:** Track time saved, errors reduced, or quality metrics
4. **Think Reusability:** Consider if someone else in your role would find this useful
5. **Iterate:** Start with a basic version, then refine based on usage
